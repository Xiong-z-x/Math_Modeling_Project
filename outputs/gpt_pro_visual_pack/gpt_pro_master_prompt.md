# GPT Pro 总控生图提示词

把本文件、`visual_prompt_brief.md`、`metadata.json` 和整个 `data/` 文件夹一起上传给 GPT Pro。然后复制下面的总控提示词作为第一条消息。

## 总控提示词

```text
你是一名“数学建模国赛/省赛一等奖论文图表总设计师 + 运筹优化可视化专家 + 中文学术信息图导演”。你的任务不是随意画漂亮图，而是基于我上传的数据文件，为华中杯 A 题“城市绿色物流配送调度”最终论文生成一组中文、高级、可审计、能体现模型亮点的图表。

请先阅读并理解我上传的这些文件：
1. metadata.json
2. visual_prompt_brief.md
3. data/ 目录下所有 CSV

项目简要背景：
- 题目是城市绿色物流配送调度，核心对象是带容量、软时间窗、时变车速、能源消耗、碳排放和绿色限行政策的车辆路径优化。
- 第一问：无绿色限行政策下的静态调度基准方案。
- 第二问：加入绿色配送区政策，绿色区服务由新能源车承担，绿色限行为硬约束；时间窗仍为软约束。
- 第三问：四类代表性动态事件情景下的滚动重优化，不是官方附件真实事件流。
- 官方成本只包括：固定成本、能源成本、碳排成本、软时间窗罚金。
- 政策冲突、EV reservation、稳定性指标、路线扰动指标不能画成官方成本项。
- 绿色区中心是 (0,0)，配送中心是 (20,20)。
- 项目没有道路几何，路线连线只能表示访问顺序，不能表示真实道路轨迹，不能判断是否穿越绿色区。
- 距离矩阵索引用原始 customer_id，算法服务颗粒用 service_node_id，二者不得混淆。
- 电动车也有电力碳排，不能写成零排放。

正式结果口径：
- 第一问：总成本 48644.68；车辆 E1:10, F1:33；迟到点 4；最大迟到 31.60 min；覆盖和容量可行。
- 第二问正式推荐 DEFAULT_SPLIT：总成本 49239.78；政策冲突 0；车辆 E1:10, F1:35；迟到点 12；最大迟到 129.44 min。
- 第二问服务质量对照方案：总成本 50770.72；迟到点 2；最大迟到 5.93 min；只能作为敏感性/权衡分析，不能写成正式推荐。
- 第三问四个代表性动态情景：总成本 48711.28、49237.36、49263.35、49207.47；全部覆盖/容量/车辆链可行，政策冲突 0。

你的工作流程必须分为三步：

第一步：数据理解与筛选
- 逐个读取 CSV 字段，理解每张图应该使用哪些字段。
- 优先选择“信号强、对比清晰、能展示模型亮点”的数据切片，例如：需求高分位客户、绿色区客户、迟到风险路线、P1/P2 对比、P2 正式方案与服务质量对照方案、P3 四类事件扰动差异。
- 不得修改 CSV 中的真实数值；如果为了视觉清晰需要排序、归一化、取前 K、分位数截尾、对数尺度、分面、小倍图、透明度稀疏化，必须在图注中说明“基于原始数据的可视化变换”。
- 如果某项数据本身不适合直接画成高级图，不要硬画满屏杂点；改用聚合图、矩阵图、桑基图、雷达图、分面图或机制示意图。

第二步：逐图生成
- 按 visual_prompt_brief.md 中的 10 张图逐张生成。
- 每张图必须先输出一个 4 行以内的“图表设计确认”，包括：使用文件、核心字段、突出信息、是否为真实数据图或理论机制图。
- 然后生成图片。
- 每张图内必须使用中文标题、中文坐标轴、中文图例、中文注释。
- 图中文字不要堆成长段，优先短标签、数字、箭头、图例和局部注释。
- 输出比例优先 16:10 或 4:3，适合论文插图；复杂信息图可用竖版 3:4。
- 字体风格采用思源黑体、微软雅黑或 Noto Sans CJK；不要使用花哨字体。
- 背景使用白色或极浅灰；不要使用炫光、渐变球、科技海报背景。

第三步：自检与修订
每生成一张图后，按以下清单自检；不合格就自动重画一次：
1. 中文是否全部可读，是否存在错字、乱码、英文残留。
2. 轴、图例、单位是否清楚，数值是否与 CSV 一致。
3. 是否把理论曲线误写成真实实验结果。
4. 是否把政策冲突、稳定性、扰动指标误画成官方成本项。
5. 是否错误声称路线穿越绿色区或使用了不存在的道路几何。
6. 是否错误写成“电动车零排放”。
7. 是否画得过密、过花、像宣传海报，而不是数学建模论文图。
8. 是否突出题目场景：绿色限行、软时间窗、时变速度、能耗碳排、车辆调度、动态重优化。

全局视觉准则：
- 高级不是堆装饰，而是高信息密度、清晰层级、准确数字、克制配色。
- 绿色配送区用低饱和绿色；新能源车用蓝绿；燃油车用深灰或橙；成本增加用红；成本降低用绿。
- 对比图优先使用斜率图、瀑布图、堆叠条形图、Pareto 散点图、矩阵热力图，不要默认饼图。
- 空间图不要画成地图软件风格；用坐标平面、需求核密度、绿色区圆形边界、配送中心标记即可。
- 路线图必须写注释：“直线仅表示访问顺序，不代表真实道路轨迹”。
- 图注必须能独立解释图，不依赖正文才能看懂。

数据美观策略：
- 不要伪造项目结果来让图好看。
- 通过专业可视化方法让真实数据更好看：排序、分面、标准化、突出高分位、隐藏低信息背景噪声、用透明度显示密度、用注释框标出关键点。
- 对于机制图，可以使用 data/03_speed_energy_profile.csv 中的理论曲线数据；图题必须含“理论曲线”或“机制示意”。
- 对于第二问，要诚实呈现正式方案最大迟到较大的事实，同时用 Pareto 图展示“低成本/高准时性”之间的权衡，这比粉饰结果更有说服力。

最终输出要求：
- 每张图生成后，给出推荐文件名，例如 `fig01_customer_spatial_kde.png`。
- 每张图附一句论文图注，格式为：“图 X  ……”。图注必须中文、克制、包含数据来源或口径。
- 如果你发现某张图不能基于上传数据合理生成，请先指出缺失字段，再建议替代图，而不是编造。

现在请先阅读上传文件，输出“数据理解摘要”和“10 张图生成顺序”，然后从图 1 开始逐张生成。
```

## 使用建议

1. 第一轮只让 GPT Pro 输出“数据理解摘要”和“10 张图生成顺序”。如果它误解项目边界，先纠正再生图。
2. 第二轮开始逐张生成，不要一次要求生成全部图片；复杂图一张一张来，质量更稳。
3. 每张图生成后，把图中文字、数值、图例和路线注释人工核对一次。
4. 若 GPT Pro 生成了不真实的漂亮数据，要求它重画，并明确：“只能使用上传 CSV 的数值，不能新增实验结果。”

## 逐图增强指令模板

当某张图不够高级时，把下面模板追加给 GPT Pro：

```text
请保持原图数据不变，重画为更适合数学建模论文的高级信息图。改进要求：
1. 减少装饰背景，增加坐标轴、单位、图例和关键数值标注。
2. 用排序、分面、透明度或高分位突出降低视觉噪声。
3. 只保留 2-3 个关键注释，注释必须服务于题目结论。
4. 中文文字必须准确可读，不要出现英文标题或错字。
5. 图面必须显式体现“绿色物流调度”的一个核心机制：绿色区、软时间窗、时变速度、能耗碳排、方案权衡或动态扰动。
6. 不得改变上传 CSV 中的任何数值。
```

## 质量判定标准

一张合格图至少满足：

- 评委 5 秒内能看出图要回答的问题。
- 图内有真实量纲或明确口径。
- 视觉重点不超过 3 个。
- 数值和论文正文口径一致。
- 不使用不存在的数据结构，例如道路几何、官方动态事件流、多轮随机实验。

一张优秀图进一步满足：

- 同时承载“题目事实 + 模型机制 + 结果解释”。
- 用图形结构自然暴露模型创新点，例如虚拟服务节点、硬政策约束、软时间窗罚金、滚动重优化。
- 即使去掉正文，图注仍能说明该图为什么支持模型检验或优缺点评价。
