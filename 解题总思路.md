## 1. 场景理解与核心难点

### 1.1 物理场景

物流公司从配送中心出发，为城市中 98 个客户点配送由 2169 条订单汇总而成的货物。车辆包含燃油车和新能源车。每个客户有软时间窗，早到会等待并产生等待成本，晚到会产生惩罚成本。车辆行驶速度随时间段变化，并服从正态分布。油耗、电耗与速度呈 U 型二次关系，并受当前载重率影响。

绿色配送区以城市中心 `(0, 0)` 为圆心，半径 10 km。配送中心位于 `(20, 20)`，不是绿色区圆心。第二问中，8:00-16:00 期间燃油车不得进入绿色配送区。

### 1.2 为什么不能直接套普通 VRP

普通 CVRP 只关心距离和载重；本题至少同时有六类耦合因素：

1. **双容量约束**：重量和体积都必须满足。
2. **软时间窗**：违反时间窗不是不可行，而是进入目标函数。
3. **时变速度**：同一路段出发时刻不同，行驶时间不同。
4. **随机速度影响能耗**：能耗函数是二次函数，不能直接把速度均值代入。
5. **载重相关能耗**：路线顺序会改变每段当前载重，进而改变能耗和碳排。
6. **绿色区政策约束**：第二问中车辆类型、访问时刻、客户空间位置三者耦合。

## 4. 时变行驶时间设计

### 4.1 FIFO 原则

时变路网必须满足 FIFO：同一路段同一方向上，先出发不应被后出发车辆“超越”。如果直接用“出发时刻速度 × 全程距离”，在跨越拥堵到顺畅的边界时可能导致到达时间递推混乱。

因此 ETA 必须用分段积分。

### 4.2 ETA 算子

输入：

- 路段距离 `d_ij`，来自距离矩阵；
- 出发时刻 `t_depart`，单位为绝对分钟。

算法：

```text
remaining = d_ij
t = t_depart
while remaining > 0:
    找到 t 所在速度时段 p
    取均值速度 μ_p
    计算当前时段剩余分钟 Δt
    当前时段最多可走 dist = μ_p * Δt / 60
    若 dist >= remaining:
        t += remaining / μ_p * 60
        remaining = 0
    否则:
        remaining -= dist
        t = 当前时段结束时刻
return t
```

路线递推时：

```text
arrival_j = ETA(distance(i, j), departure_i)
wait_j = max(earliest_j - arrival_j, 0)
late_j = max(arrival_j - latest_j, 0)
departure_j = max(arrival_j, earliest_j) + 20
```

注意：惩罚基于 `arrival_j`，服务开始时间用于递推后续节点。

## 5. 能耗与碳排设计

### 5.1 Jensen 修正

题面给出速度服从正态分布，而油耗和电耗是二次函数。若直接代入均值速度，会系统性低估二次项。

燃油车：

```text
FPK(v) = 0.0025 v^2 - 0.2554 v + 31.75
E[FPK(V)] = 0.0025 (μ^2 + σ^2) - 0.2554 μ + 31.75
```

新能源车：

```text
EPK(v) = 0.0014 v^2 - 0.12 v + 36.19
E[EPK(V)] = 0.0014 (μ^2 + σ^2) - 0.12 μ + 36.19
```

这是本文的第一个创新点：在赛题给定随机速度的前提下，不把随机性粗暴压成均值，而是保留二阶矩对能耗的影响。

### 5.2 跨时段能耗积分

行驶时间跨时段时，能耗也必须按时段拆分，而不是整段用出发时段能耗。

对于路段 `(i, j)`，按照 ETA 积分过程得到若干子段：

```text
(dist_1, period_1), (dist_2, period_2), ...
```

每个子段分别计算能耗：

```text
consumption_segment = dist_s / 100 * expected_rate(period_s) * load_factor
```

其中当前载重率：

```text
ρ = current_weight / vehicle_capacity_weight
```

载重修正：

```text
燃油车 load_factor = 1 + 0.40 * ρ
新能源 load_factor = 1 + 0.35 * ρ
```

### 5.3 成本换算

燃油车：

```text
fuel_cost = liters * 7.61
carbon_kg = liters * 2.547
carbon_cost = carbon_kg * 0.65
```

新能源车：

```text
electricity_cost = kWh * 1.64
carbon_kg = kWh * 0.501
carbon_cost = carbon_kg * 0.65
```

## 6. 求解算法选择

### 6.1 三种路线比较

| 路线 | 优点 | 缺点 | 结论 |
| --- | --- | --- | --- |
| 精确 MILP / MINLP | 模型严谨 | 148 个虚拟节点、多车型、时变速度、软时间窗导致规模过大 | 用于论文建模，不作为主求解器 |

采用 ALNS 是因为本题属于多属性 VRP。Pisinger 和 Ropke 的 ALNS 框架可处理多类 VRP 变体，污染路径问题文献也表明 ALNS 适合速度、油耗、排放共同优化的路径问题。

```text

```

## 7. 初始解策略

初始解不追求极优，但必须可评价、容量合法，并尽量少产生极大惩罚。

推荐流程：

1. 按时间窗中心或最早开始时间对虚拟服务节点排序。
2. 先处理需求大、时间窗紧的节点。
3. 优先尝试 F1/E1 大车插入，以减少车辆数和拆分客户分散程度。
4. 对每个节点枚举已有路线中的插入位置。
5. 插入后完整重算路线时序和成本。
6. 若没有合法插入，则开启新路线。

初始解构造时的车选择：

- 第一问：以成本和容量为主，不强制 EV。
- 第二问：绿区节点优先分配 EV；若 EV 容量不足，则允许燃油车在 16:00 后服务。

## 8. ALNS 破坏与修复算子

### 8.1 破坏算子

1. **Random Remove**：随机移除若干虚拟节点，保证多样性。
2. **Worst Cost Remove**：移除边际成本贡献最大的节点。
3. **Related Remove**：移除空间近、时间窗近、同客户拆分节点相关的一组节点。
4. **Time Penalty Remove**：优先移除造成大额早到等待或迟到惩罚的节点。
5. **Split-Customer Remove**：针对同一原客户的多个虚拟节点成组调整，减少拆分节点被过度分散。
6. **Green-Zone Remove**：第二问中优先移除违规或接近违规的绿区节点。

### 8.2 修复算子

1. **Greedy Insert**：选择成本增量最小的位置。
2. **Regret-2 / Regret-3 Insert**：优先插入“现在不插以后更贵”的节点。
3. **Time-Oriented Insert**：对紧时间窗节点按时间窗中心优先插入。
4. **Vehicle-Upgrade Insert**：当小车容量限制导致路线破碎时，尝试升级车型。
5. **EV-Priority Insert**：第二问中绿区节点优先插入新能源路线。

### 8.3 接受准则

采用模拟退火：

```text
若 new_cost < current_cost: 接受
否则以 exp(-(new_cost-current_cost)/T) 接受
T = αT
```

算子权重按阶段得分更新：

- 产生全局最优：高分；
- 改善当前解：中高分；
- 被接受但未改善：低分；
- 被拒绝：不加分。

## 10. 第二问技术路线

第二问不是从零建模，而是在第一问评价器上加入政策约束。

### 10.1 政策判定

燃油车访问绿区客户时：

```text
若 arrival_time ∈ [08:00, 16:00]，则违规
```

由于题面说燃油车不得进入绿色配送区，严格做法还应考虑路段穿越绿区。但附件只给客户坐标和节点间路网距离，没有道路几何轨迹。因此可操作建模采用：

> 若燃油车在限行时段服务绿区客户，则视为进入绿区违规。

该假设要在论文中说明。

### 10.2 求解策略

1. 以第一问最优或较优解为初始解。
2. 找出燃油车限行时段服务绿区客户的路线。
3. 优先将绿区节点迁移到 EV 路线。
4. 若 EV 数量或容量不足，允许燃油车在 16:00 后服务绿区。
5. 对受影响路线运行局部 ALNS，不必完全推翻非违规路线。

在事件时刻 `t_now`：

- 已完成服务的节点冻结；
- 正在路上的车辆当前位置和载重冻结；
- 尚未出发或尚未服务的后续节点可重排；
- 已经承诺即将到达的客户可设置较高变更惩罚，避免方案过度跳变。

补充物理规则：已出发趟次中的未服务货物已经在车上，不能无条件转移给其他车辆；新增订单也不能凭空插入未回库车辆。若要允许这些动作，必须显式建模回库、换装或交叉转运。MVP 阶段优先重排未发车趟次和回库后趟次。

### 11.3 动态求解流程

```text
读取当前方案
识别冻结路线片段与可调整节点
应用事件变更
构建局部子问题
用快速 repair + 轻量 ALNS 重优化
输出调整前后对比
```

### 11.4 建议展示场景

1. 10:30 订单取消：删除尚未服务节点，尝试合并路线节约成本。
2. 13:30 新增绿区订单：优先插入 EV 或 16:00 后燃油路线。
3. 15:00 时间窗提前：局部重排并展示惩罚变化。
4. 12:00 地址变更：使用既有客户点作为代理地址，避免虚构新距离矩阵。

## 12. 创新点设计

### 12.1 Jensen 修正的期望能耗

本题明确给出速度正态分布。能耗是凸二次函数，因此必须使用 `E[v^2] = μ^2 + σ^2`。这比直接代入均值速度更严谨。

### 12.2 时间和能耗双重分段积分

多数简化方案只对时间做分段，能耗仍用出发时段速度。本文要求二者同时分段，避免跨时段路段成本失真。

### 12.3 数据驱动的绿区重判定

原题正文提到 30 个绿区客户，但补充说明要求按坐标重新计算。本项目以附件坐标计算得到 15 个绿区客户、12 个有效绿区客户，并让代码测试固定这一事实。

### 12.4 虚拟节点但保留客户映射

聚合客户后需求超大，必须拆分。但距离矩阵只有原始客户 ID。本文采用虚拟服务节点 + `node_to_customer` 映射，既保证容量可行，又不破坏原距离矩阵。

### 12.5 路线池精修

ALNS 后可选增加路线池 set partitioning 精修：

1. 保存搜索过程中出现的高质量路线；
2. 构建覆盖所有虚拟节点一次的集合划分模型；
3. 用轻量整数规划选择路线组合。

这不是第一版必须项，但可作为论文加分点和后续性能增强。

### 12.6 鲁棒性验证

最终方案可用蒙特卡洛采样速度扰动验证：

- 固定路线结构；
- 多次采样各时段速度；
- 统计总成本、迟到风险、碳排波动；
- 对比 Jensen 期望模型与仿真均值。

## 14. 性能设计

### 14.1 缓存

高频函数必须缓存：

- `ETA(distance, depart_slot)`；
- `segment_split(distance, depart_slot)`；
- `route_cost(route_signature, vehicle_type)`；
- 节点间相关性矩阵；
- 最近邻候选表。

时间可按 1 分钟或 5 分钟离散缓存。第一版建议 5 分钟离散以提高速度，最终论文结果可用精确分钟重算。

### 14.2 候选插入限制

148 个虚拟节点下，全量插入仍可接受，但 ALNS 长迭代会慢。修复算子可使用候选邻域：

- 空间最近 20 个客户；
- 时间窗中心相近；
- 同一原客户拆分节点优先靠近；
- 对绿区节点限制候选车辆类型。

### 14.3 增量评价

插入或移除节点后，只重算受影响路线，而不是全解重算。最终每若干轮做一次全量一致性校验。

## 15. 验证与质量门

### 15.1 单元测试

必须覆盖：

1. 数据事实；
2. 速度时段查询；
3. 跨时段 ETA；
4. FIFO 单调性；
5. Jensen 修正值；
6. 载重修正；
7. 时间窗惩罚；
8. 路线容量；
9. 绿区违规判定；
10. 输出格式。

### 15.2 集成测试

1. 构造 3-5 个节点小实例，手工核算路线成本。
2. 用真实数据运行快速初始解。
3. 用真实数据运行短迭代 ALNS。
4. 检查所有虚拟节点被服务一次。
5. 检查所有路线容量合法。

### 15.3 结果可信性检查

1. 车辆数是否低于总需求理论下界过多，若是则容量检查可能有 bug。
2. 时间窗惩罚是否异常大，若是初始解或 ETA 可能有问题。
3. EV 使用是否在第一问中过度偏高，若是成本参数可能有误。
4. 第二问是否仍有燃油车限行时段服务绿区，若有则政策约束失败。

## 18. 当前最终决策

1. 主求解器采用 ALNS。
2. 数据层以 148 个虚拟服务节点为算法节点。
3. 时间使用绝对分钟。
4. 行驶时间用均值速度分段积分。
5. 能耗用 Jensen 修正后的期望二次能耗，并按跨时段子段累计。
6. 第一问先求高质量静态基准解。
7. 第二问在第一问基础上加入绿区限行约束并做对比。
8. 第三问采用滚动时域局部重优化。
9. 所有核心模块必须测试先行。
